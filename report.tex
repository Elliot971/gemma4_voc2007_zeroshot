\begin{abstract}
视觉语言模型VLM的快速发展正在改变目标检测的范式。传统检测器依赖大量标注数据进行有监督训练，而 VLM 通过预训练阶段积累的视觉-语言对齐能力，有望实现零标注成本的目标检测。我在 Pascal VOC 2007 上进行三范式对比评测：(1) 有监督专用检测器 PLN-ResNet18，(2) Gemma-4 E2B 零样本检测，(3) Gemma-4 E2B LoRA 微调后检测。

三个阶段使用同一数据集VOC 2007 Test、同一评测指标和同一份 \texttt{voc\_eval.py} 评测代码。PLN-ResNet18 取得 68.74\% mAP@0.5。Gemma-4 E2B 零样本推理取得 \textbf{51.51\%} mAP@0.5和30.05\%mAP@0.75。我通过 LoRA 微调 Gemma-4 的视觉编码器和语言模型，微调后 mAP@0.5 达到 \textbf{67.58\%}、mAP@0.75 达到 51.00\%，仅比 PLN 低 1.16 个百分点。

将三个模型的结果对比我发现：(1) 零样本检测在语义特征鲜明的大物体上，比如cat有90.89\%，超越专用检测器；(2) LoRA 微调在弱类别比如diningtable、bottle、boat等上带来显著提升；(3) 三个不同形式的模型在能力维度上呈互补特征：有监督擅长精确定位，零样本擅长语义理解，微调在两者间寻求平衡。
\end{abstract}

\section{引言}

\subsection{任务背景}

在上一个任务中，我复现了 Point Linking Network，将 Backbone 从 Inception-v2 替换为 ResNet-18，在 VOC 2007 Test 上取得 68.74\% mAP@0.5。第二阶段的任务是：使用 Gemma 4 最小模型，在同一数据集上评测零样本 mAP。

两个阶段使用同一数据集、同一评测指标、同一份评测代码，目的是希望\textbf{在同一把尺子下量化"有监督专用检测"与"零样本通用检测"的差距}，建立对目标检测范式演进的直观认知。

\subsection{对比的意义}

\begin{itemize}
    \item \textbf{PLN-ResNet18}（有监督）：68.74\% mAP，165M 参数，$\sim$30ms/张
    \item \textbf{Gemma-4 E2B}（零样本）：51.51\% mAP，2.3B 参数，$\sim$4.4s/张
    \item \textbf{Gemma-4 E2B + LoRA}（微调）：67.58\% mAP，+169M 可训练参数，$\sim$6.6s/张
\end{itemize}

三种方式代表了目标检测领域的不同路径：精确拟合、零标注泛化、高效特化。量化它们之间的差距，有助于理解 VLM 在视觉任务中的能力边界和适用场景。

\section{方法设计}

\subsection{Prompt 设计}

设计要点：

\begin{enumerate}
    \item \textbf{显式列出全部 20 类}：避免模型遗漏类别或输出非 VOC 类别名称。
    \item \textbf{指定 \texttt{box\_2d} 格式}：这是 Gemma 4 原生训练的检测输出格式（1000$\times$1000 归一化坐标空间），使用模型训练时的原生格式可以最大化检测质量。
    \item \textbf{要求纯 JSON 输出}：减少解析复杂度。实际测试中模型仍会包裹 markdown code fence（\texttt{```json ... ```}），需在解析前去除。
\end{enumerate}

\subsection{输出解析}

模型的自回归文本输出需要多级处理转化为标准检测框。

\subsubsection{Markdown fence 去除}

Gemma 4 将 JSON 包裹在 \texttt{```json ... ```} 中。使用正则表达式在 JSON 解析前去除：

\begin{verbatim}
text = re.sub(r"^```(?:json)?\s*\n?", "", text.strip())
text = re.sub(r"\n?```\s*$", "", text.strip())
\end{verbatim}

\subsubsection{三级容错 JSON 解析}

\begin{table}[H]
\centering
\caption{三级容错解析策略}
\label{tab:parsing}
\begin{tabular}{@{}clc@{}}
\toprule
\textbf{级别} & \textbf{策略} & \textbf{实际命中率} \\
\midrule

L1 的高命中率说明 Gemma 4 E2B 的指令遵循能力已经相当稳定。三级设计是防御性编程，确保即使模型输出格式异常也能最大限度地提取有效信息。

\subsubsection{坐标转换}

Gemma 4 输出 \texttt{box\_2d} = $[y_{\min}, x_{\min}, y_{\max}, x_{\max}]$，坐标空间为 0--1000 归一化整数，需转换为 VOC 标准的 $[x_{\min}, y_{\min}, x_{\max}, y_{\max}]$ 像素坐标：

\begin{equation}
    x_{\text{pixel}} = \frac{x_{1000}}{1000} \times W, \quad
    y_{\text{pixel}} = \frac{y_{1000}}{1000} \times H
\end{equation}

转换后对框进行合法性校验，丢弃退化框。

\subsubsection{类别模糊匹配}

VLM 输出的类别名可能与 VOC 标准名不一致。通过三级匹配解决：

\begin{enumerate}
    \item \textbf{精确匹配}：\texttt{dog} $\to$ \texttt{dog}
    \item \textbf{别名表查找}：维护 30+ 条映射，如 \texttt{airplane} $\to$ \texttt{aeroplane}，\texttt{motorcycle} $\to$ \texttt{motorbike}，\texttt{tv} $\to$ \texttt{tvmonitor}
    \item \textbf{子串匹配兜底}：如 \texttt{dining table} $\to$ \texttt{diningtable}
\end{enumerate}

\subsection{置信度评估}

传统检测器直接输出每个框的 confidence score。VLM 的自回归生成没有这一概念，本项目使用 \textbf{token log-probability 的几何平均值}作为序列级置信度：

\begin{equation}
    \text{confidence} = \exp\left(\frac{1}{T} \sum_{t=1}^{T} \log p(x_t \mid x_{<t})\right)
\end{equation}

其中 $T$ 为生成的 token 数量。该分数反映模型对整段输出的"确定性"，用于 VOC mAP 计算中 precision-recall 曲线的排序。

需要注意的是，这是一个\textbf{序列级}而非\textbf{框级}的置信度——同一张图上的所有检测框共享同一个 score。这是 VLM 做检测的固有限制：框是序列生成的一部分，无法为每个框独立打分。

\section{微调方法}

零样本检测虽然展示了 VLM 的泛化能力，但 51.51\% 的 mAP 与有监督检测器仍有 17.23 个百分点的差距。我通过 LoRA 微调缩小这一差距。

\subsection{微调方案}

微调对我的核心挑战在于：如何在有限计算资源下高效适配一个 5B 参数级的 VLM 到 VOC 目标检测任务。

\begin{table}[H]
\centering
\caption{LoRA 微调配置}
\label{tab:lora_config}
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{配置项} & \textbf{选择} & \textbf{理由} \\
\midrule
微调方法 & LoRA (非 QLoRA) & 4090 24GB 支持 bf16 \\
LoRA rank & 64 & 适中，平衡效果和参数量 \\
LoRA alpha & 64 & alpha = rank，不额外缩放 \\
LLM target & q/v proj, gate/up/down proj & 注意力 + FFN 全覆盖 \\
视觉编码器 & ClippableLinear patch & 恢复细粒度空间特征 \\
学习率 & $2 \times 10^{-4}$ & LoRA 常用学习率 \\
Batch size & 1 $\times$ 16 grad\_accum & 等效 16 \\
Epochs & 3 & loss 收敛后停止 \\
权重衰减 & 0.1 & 防过拟合 \\
LR Scheduler & Cosine & 小数据泛化好 \\
\bottomrule
\end{tabular}
\end{table}

\subsection{数据准备}

微调数据来自 VOC 2007 + 2012 的 trainval 集，共有16,551 张，46,549 个标注框，加上水平翻转增强后共 24,802 条样本。每条样本采用以下格式：

\begin{verbatim}
User: Detect all objects in this image: aeroplane, bicycle, ...
Model: [{"box_2d": [y_min, x_min, y_max, x_max], "label": "cat"}, ...]
\end{verbatim}

使用 10 种不同的 prompt 措辞随机选用，防止模型过拟合到特定句式。标注数据中偶有 \texttt{xmax > img\_w} 的标注误差，在转换时进行了 clamp 处理。



\subsection{训练过程}

回顾整个微调过程，共遇到 6 个关键工程问题。

\textbf{（1）Gemma4ClippableLinear 与 PEFT 不兼容}

Gemma 4 视觉编码器使用自定义层 \texttt{Gemma4ClippableLinear}，PEFT 不支持此类型。

编写 \texttt{\_patch\_clippable\_linear()} 函数：遍历模型找到所有 \texttt{Gemma4ClippableLinear} 实例，创建 \texttt{PatchedClippableLinear} 子类，继承 \texttt{nn.Linear}，保留原始权重和 clamp 行为。关键细节是必须保持原始 dtype 和 device，否则推理时报 dtype mismatch。此外，使用全限定名作为 LoRA target modules，避免 PEFT 按 leaf name 误匹配。

这个 patch 在三个地方使用：\texttt{finetune\_gemma.py}、\texttt{eval\_voc.py}、以及加载 adapter 后的 \texttt{model.to("cuda").to(torch.bfloat16)}。

\textbf{（2）视觉编码器 LoRA target 收集困难}

Gemma 4 视觉编码器的 Linear 层命名不规范，PEFT 的 \texttt{target\_modules} 按 leaf name 匹配会误匹配到非目标层。

编写 \texttt{\_get\_target\_modules()} 函数，手动收集所有符合条件的模块的全限定路径（如 \texttt{vision\_tower.vision\_model.vision\_encoder.layers.0.mlp.gate\_proj}），直接传给 PEFT，避免模糊匹配。

\textbf{（3）训练数据格式与 Processor 批处理不兼容}

\texttt{Gemma4Processor} 的 \texttt{apply\_chat\_template} 对批处理时 \texttt{images} 参数要求嵌套列表 \texttt{[[img1], [img2]]}，单张图直接用 \texttt{[img1]} 会报错。

在 \texttt{VLMDataCollator} 中构造正确的嵌套格式，并处理单样本情况。

\textbf{（4）坐标空间约定}

Gemma 4 原生使用 $[0, 1000]$ 整数坐标空间，但 VOC XML 标注偶有 $x_{\max} > \text{img\_w}$ 的标注误差。

在 \texttt{prepare\_finetune\_data.py} 中对坐标进行 clamp 到 $[0, 1000]$，保持整数格式，不引入小数改变模型原生约定。

\textbf{（5）训练效率与显存平衡}

5B 参数模型在 24GB 显存上全量训练不可行。
\begin{itemize}
    \item LoRA：bf16 精度，可训练参数 169M
    \item batch size=1 + grad\_accum=16，等效 batch=16
    \item cosine scheduler + warmup
    \item 3 epoch，约 13 小时完成
\end{itemize}

初始尝试 batch size=2 时在反向传播阶段触发 OOM，说明 24GB 显存对 5.3B 参数 bf16 训练态已是物理上限。

\textbf{（6）推理时加载 LoRA adapter 的 device/dtype 一致性}

加载 adapter 后模型 dtype 变成 float32，推理时报错。

加载 adapter 后显式调用 \texttt{model = model.to("cuda").to(torch.bfloat16)}。

\textbf{训练过程}。总步数 4,653 步，使用 AdamW 优化器。可训练参数 168.6M（占总参数 5.3B 的 3.2\%）。训练 loss 从初始的 2.202 快速下降到最终的 0.276，降幅达 87.5\%，呈稳步下降趋势，无过拟合迹象。总训练时间约 13 小时。

\subsection{类别分析}

\textbf{第一梯队（AP $>$ 60\%）}：cat (90.89\%), dog (82.70\%), train (70.55\%), horse (66.61\%), cow (65.11\%), bird (63.71\%), bus (61.72\%), bicycle (61.19\%)。共性特征是物体通常较大、形态特征鲜明、在互联网图像中出现频率高。cat 的 90.89\% 说明 VLM 对这类高辨识度目标的检测能力已经非常强。

\textbf{第二梯队（AP 40--60\%）}：motorbike (59.86\%), sheep (57.84\%), person (53.30\%), car (51.87\%), aeroplane (50.55\%)。person 虽然是最高频类别，但 AP 仅 53.30\%，主要原因是密集场景下 VLM 难以逐个输出所有实例。

\textbf{关键发现}：

\begin{enumerate}
    \item \textbf{定位误差（12.4\%）不是主要问题}。这些框检测到了正确的物体类别，但框的精度不足，IoU 在 0.1--0.5 之间。如果 IoU 阈值放宽到 0.1，大部分会变为 TP。
    \item \textbf{背景误检（16.8\%）是最大的 FP 来源}。2,220 个检测框与所有 GT 的 IoU 均低于 0.1，本质上是"幻觉"——模型生成了场景中不存在的物体框。其中 chair、sofa和 diningtable是重灾区，说明模型对室内家具类物体的边界判断极不稳定。
    \item \textbf{漏检（29.7\%）是 mAP 损失的主因}。3,576 个 GT 框未被检测到。漏检最严重的类别是 person,有1278 个漏检，占 person GT 的 28.2\%、car和 pottedplant。这与自回归生成的"提前终止"倾向一致——密集实例场景中，模型倾向于只输出最显著的目标。
    \item \textbf{sofa 的反常表现}：sofa 的召回率高达 89.1\%，但 AP 只有 36.24\%。原因是 sofa 产生了 361 个背景误检，大量误检严重拉低了 precision。
\end{enumerate}

\textbf{零样本接近或超越 PLN 的类别}：cat ：+0.90, dog ：$-$0.85。cat 是唯一零样本超过 PLN 的类别，dog 几乎持平。这两个类别在互联网图像中的出现频率极高，VLM 在预训练阶段积累了大量相关视觉-语义知识。

\textbf{微调后大幅改善的类别}：LoRA 微调后，FT 在 9/20 个类别上超过 PLN，包括 cat ：92.52\%, bird ：78.09\%, boat ：64.54\%, aeroplane ：74.20\%, bus ：79.68\%, cow ：74.23\%, person：69.08\%, pottedplant：38.77\%。这些类别在零样本阶段都是 PLN 大幅领先的，说明微调有效弥补了 VLM 在边界定义和空间定位上的短板。

PLN 的核心优势在于\textbf{精确的空间定位能力}：通过关键点检测和链接回归，直接在特征图上学习坐标映射。这种能力是通过在 16k+ 张标注图像上迭代数万次获得的。

Gemma-4 零样本的核心优势在于\textbf{强大的语义理解}：不需要看过任何 VOC 标注就"知道" cat 长什么样，并能给出 90.89\% 的 AP。这种能力来自大规模视觉-语言预训练中积累的通用视觉知识。

Gemma-4 LoRA 微调则试图\textbf{在两者间寻求平衡}：保留 VLM 的语义理解能力，同时通过在 VOC 训练集上微调视觉编码器和 LLM 部分权重，提升空间定位精度和 VOC 特定类别的检测能力。可训练参数仅占总参数的 3.2\%（169M / 5.3B），是典型的高效微调。

有监督和零样本之间17.23 个 mAP 点的差距，本质上是"通用智能的零样本泛化"与"任务特化的精确拟合"之间的 trade-off。微调的结果将填补这一差距的多少，取决于视觉编码器参数的可迁移性以及 LoRA 的表达能力。

\subsection{微调提升分析}

微调带来了 \textbf{+16.07 个百分点}的 mAP 提升，几乎追平 PLN 的 68.74\%。

\begin{itemize}
    \item \textbf{提升最大的类别}：boat (+39.68), tvmonitor (+29.39), sofa (+26.65), diningtable (+23.42), aeroplane (+23.65), pottedplant (+21.91)。这些正是零样本 mAP 最低的类别，瓶颈在于 VLM 对物体边界定义不清。LoRA 微调视觉编码器直接解决了这一问题。
    \item \textbf{提升有限的类别}：cat (+1.63), dog (+0.62)。零样本已达 82-91\% AP，接近饱和，微调的边际收益有限。
    \item \textbf{FT 超过 PLN 的类别（9/20）}：cat (92.52 vs 89.99), bird (78.09 vs 70.46), boat (64.54 vs 48.67), aeroplane (74.20 vs 70.54), bus (79.68 vs 78.24), cow (74.23 vs 70.67), person (69.08 vs 66.49), pottedplant (38.77 vs 35.61)。
    \item \textbf{PLN 仍然领先的类别}：bicycle (81.72 vs 74.55), car (76.30 vs 73.02), horse (83.19 vs 82.29), motorbike (79.28 vs 73.24), sheep (73.71 vs 69.95), train (83.32 vs 78.90), dog (83.55 vs 83.32)。PLN 对密集检测和精确定位仍有优势。
    \item \textbf{推理速度}：微调后推理速度 6.62s/张，比零样本的 4.37s/张慢 51.5\%，这是因为 LoRA adapter 增加了额外的前向传播开销。
\end{itemize}

\section{讨论}

\subsection{mAP 在 VOC 历史脉络中的位置}

将三个范式的 mAP 放在 VOC 2007 检测的历史脉络中：

\begin{itemize}
    \item 随机基线（均匀猜测 + 随机框）：约 1--2\% mAP
    \item DPM v5（2010, 传统特征）：33.7\% mAP
    \item R-CNN（2014, 深度学习开端）：58.5\% mAP
    \item \textbf{Gemma-4 E2B（2025, 零样本）}：\textbf{51.51\% mAP}
    \item \textbf{Gemma-4 E2B（2025, LoRA 微调）}：\textbf{67.58\% mAP}
    \item PLN-ResNet18（2017 架构, 有监督）：68.74\% mAP
    \item Faster R-CNN + ResNet-101（2016, 有监督）：76.4\% mAP
\end{itemize}

一个 2.3B 参数的通用模型，在零样本条件下达到介于 DPM v5 和 R-CNN 之间的性能，经 LoRA 微调后（仅训练 3 epoch、13 小时）预期进一步提升。这描绘了 VLM 目标检测的演进路径：从零样本泛化到高效微调特化。

\subsection{瓶颈分析}

\textbf{空间定位精度}：VLM 生成的坐标是离散化到 1000$\times$1000 网格的整数，理论最大精度约 0.1\%。但实际瓶颈不在此——2.3B 模型的视觉编码器分辨率有限，对小目标的定位误差远大于网格精度。IoU 阈值敏感性分析（表~\ref{tab:iou_sensitivity}）直接验证了这一点：mAP 从 0.5 到 0.75 阈值下降 41.7\%，远高于有监督检测器的典型降幅（15--25\%）。错误分析进一步表明，12.4\% 的检测框属于"定位误差"类型（正确类别但 IoU 不足），如果框的精度能提升到 IoU$\geq$0.5，mAP 将显著改善。

\textbf{密集实例检测}：当一张图中同类物体超过 3--4 个时（如一群人、一排瓶子），模型倾向于只输出最显著的几个。错误分析（表~\ref{tab:gt_recall}）显示 29.7\% 的 GT 框未被检测到，其中 person 类漏检 1,278 个（占 person GT 的 28.2\%），是漏检最严重的类别。这是 VLM 自回归生成范式的固有限制——每多输出一个框就多消耗数十个 token，而模型在生成过程中会倾向于"及时停止"。

\textbf{推理效率}：4.37s/张的速度使其无法用于任何实时场景。这是 VLM 做检测的根本 trade-off：获得了零样本泛化能力，但牺牲了两个数量级的推理效率。在自动驾驶等对延迟敏感的场景中，这一差距目前是不可接受的。

\section{结论}

报告中总结了我完成了三项工作：Gemma-4 E2B 在 Pascal VOC 2007 test 集上的零样本目标检测评测，LoRA 微调后检测评测，以及与 PLN-ResNet18 的三范式对比分析。主要结论如下：

\begin{enumerate}
    \item \textbf{零样本检测达到可用水平}：mAP@0.5 = 51.51\%，mAP@0.75 = 30.05\%，4952 张图推理成功率 99.5\%，仅 1 张解析失败。
    \item \textbf{LoRA 微调后 mAP@0.75 大幅提升}：从 30.05\% 提升到 51.00\%，直接提升了20.95 个百分点，降幅从 41.7\% 收窄到 24.5\%，证明微调显著改善了空间定位精度。
    \item \textbf{LoRA 微调大幅提升检测能力}：微调后 mAP@0.5 = 67.58\%（+16.07 个百分点），仅用 3.2\% 的可训练参数（169M / 5.3B），训练 3 epoch 约 13 小时，与 PLN 的 68.74\% 仅差 1.16 个点。在 9/20 个类别上超过 PLN。
    \item \textbf{微调最大受益类别是零样本弱势类别}：boat (+39.68), tvmonitor (+29.39), sofa (+26.65), diningtable (+23.42), aeroplane (+23.65)。而 cat (+1.63)、dog (+0.62) 等已接近饱和的类别提升有限。
    \item \textbf{空间定位通过微调显著改善}：零样本的核心短板——边界模糊、幻觉误检通过 LoRA 微调视觉编码器得到直接解决。
    \item \textbf{三种范式互补而非替代}：有监督擅长精确定位和高效推理，零样本擅长语义理解和零标注泛化，微调在两者间取得平衡——保留 VLM 的语义理解，同时通过 VOC 数据特化定位能力。
\end{enumerate}

三个阶段的实验数据说明了一个明确的趋势：VLM 的零样本能力已经达到可用水平，有51.51\% mAP，而通过3.2\% 参数的轻量级微调，13 小时训练即可接近专用检测器的性能（67.58\% vs 68.74\%）。对于自动驾驶、具身智能等需要快速适配新场景的领域，"通用模型 + 高效微调"的路径可能比"为每个任务训练专用模型"更具 practical value——前提是推理效率的瓶颈能够得到解决。
